2. Project Overview

What It Does

Morpho is a permissionless lending protocol that allows anyone to create isolated lending markets and earn yield. It operates on two layers:

  1. Morpho Blue — Immutable base layer for creating isolated lending markets
  2. Morpho Vaults — Curated vaults that aggregate liquidity across markets for easier UX

Key Innovation

  • Isolated markets — Each market has one loan asset, one collateral asset, one oracle, one LLTV. Risks are contained.
  • Immutable contracts — Cannot be changed after deployment, reducing governance risk
  • Better rates — Direct lender-borrower matching narrows the spread vs. traditional pools

3. How It Works (User Perspective)

High-Level Explanation

  1. Lenders deposit assets into Morpho Vaults (curated) or directly into Morpho Markets (advanced)
  2. Borrowers deposit collateral and borrow against it in isolated markets
  3. Curators create and manage vaults that aggregate markets with similar risk profiles
  4. Market Creators can permissionlessly deploy new lending markets

User Roles & Incentives

Lenders

  • Actions: Deposit into vaults or markets
  • Incentives: Earn interest from borrowers + potential MORPHO rewards
  • Risks: Smart contract risk, oracle risk, borrower defaults (isolated per market)

Borrowers

  • Actions: Deposit collateral, borrow assets
  • Incentives: Access liquidity, higher LTVs than competitors
  • Risks: Liquidation if collateral value drops

Curators (Vault Managers)

  • Actions: Create vaults, select which markets to include, set allocation
  • Incentives: Can charge performance fees, build reputation
  • Risks: Reputational risk if vault underperforms

Governance Participants

  • Actions: Vote on protocol parameters, fee switch, token distributions
  • Incentives: Shape protocol direction
  • Risks: Low participation = centralized decisions
